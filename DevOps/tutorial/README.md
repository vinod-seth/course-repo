## 🎯 Quick Start: Choose Your Path

### Track A: Non-Technical (Business, Leadership)
**Time:** 20–35 hours over 4–5 weeks  
**Goal:** Understand DevOps strategy, governance, risk, cost

**Your Learning Path:**
1. Module 1: DevOps Fundamentals (45 min) — Why DevOps matters
2. Module 2: Version Control Workflows (60 min) — Code review gates
3. Module 3: GitHub Actions (60 min) — Understand workflow triggers
4. Module 4: Azure Pipelines (60 min) — Understand stages & approval
5. Module 5: CI — Build & Test (75 min) — What testing means
6. Module 6: CD — Deployment (75 min) — Deployment strategies & risk
7. Module 7: Approval Gates (75 min) — Who approves & why
8. Module 8: Secrets & Security (90 min) — Compliance & secrets
9. Module 9–10: Artifacts & Monitoring (120 min) — Versioning & health
10. Module 11: Scaling Patterns (75 min) — Real-world challenges
11. **Capstone Scenario Walkthrough** (90 min) — End-to-end narrative

**Assessment:** Scenario-based questions + capstone walkthrough comprehension

**What You Can Do After:**
- Interpret a pipeline and explain its stages to engineers
- Decide on deployment strategies (blue-green vs. canary)
- Evaluate cost vs. governance trade-offs
- Read DORA metrics and assess team health
- Speak with confidence about DevOps with technical teams

### Track B: Technical (Developers, DevOps Engineers)
**Time:** 60–120 hours over 8–10 weeks  
**Goal:** Implement and troubleshoot CI/CD pipelines end-to-end

**Your Learning Path:**
1. **Modules 1–2:** Foundations (2 hours)
2. **Modules 3–4:** Learn both platforms (hands-on labs, 8 hours)
3. **Module 5:** CI pipeline (build, test, coverage, labs, 6 hours)
4. **Module 6:** CD pipeline (blue-green, canary, labs, 6 hours)
5. **Module 7:** Approval gates and environments (labs, 4 hours)
6. **Module 8:** Secrets & OIDC (hands-on, 4 hours)
7. **Modules 9–10:** Artifacts, versioning, monitoring (labs, 4 hours)
8. **Module 11:** Scaling patterns (discussion, 2 hours)
9. **Capstone:** Full implementation (4–6 hours)

**Labs You'll Complete:**
- ✅ Create a GitHub Actions workflow for multi-language projects
- ✅ Build a multi-stage Azure DevOps pipeline
- ✅ Set up approval gates and environments
- ✅ Configure secrets and OIDC authentication
- ✅ Implement blue-green deployments
- ✅ Create health checks and monitoring dashboards
- ✅ **Capstone:** Build an end-to-end pipeline (checkout feature or tenant onboarding)

**Assessment:** Working pipelines + troubleshooting tasks + capstone project

**What You Can Do After:**
- Design and implement CI/CD pipelines for production
- Write YAML workflows and pipelines
- Set up multi-stage deployments with gates
- Manage secrets and compliance requirements
- Troubleshoot pipeline failures
- Optimize for speed and cost
- Guide teams on DevOps best practices

- Pipeline strategy: Polyrepo, Azure DevOps for CI/CD
- Deployment: Canary (limited risk)
- Featured in: Modules 1, 2, 4, 7, 8, 9, 10, 11

## 📞 Questions & Support

### Common Questions

**Q: Do I need to know Git?**
- Track A: No, basics are taught in Module 2
- Track B: Yes, should be comfortable with commit/push/branch

**Q: Do I need to know Docker?**
- Track A: No, Docker is explained conceptually
- Track B: Helps but not required; we explain as we go

**Q: Do I need an Azure subscription?**
- Track A: No
- Track B: Yes, free tier works (create free account at azure.microsoft.com)

**Q: How long should each module take?**
- Track A: 45 min – 90 min per module
- Track B: 2 – 4 hours per module (concept + lab)

**Q: Can I skip modules?**
- Track A: Recommended to follow order (concepts build on each other)
- Track B: Can skip some theory if experienced, but labs should be sequential

**Q: Are there any prerequisites?**
- Track A: Basic understanding of software delivery
- Track B: Intermediate software development experience + Git basics
