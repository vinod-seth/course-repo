## 🟢 Section 1: Core CI/CD & Pipeline Engineering

**Question 1:** Which component in GitHub Actions defines the runner operating system and trigger events?
* [ ] Action file (.js)
* [x] Workflow YAML file (.github/workflows/*.yml)
* [ ] Environment secret
* [ ] Deployment gate

<details>
<summary>🔑 Click to Reveal Answer & Explanation</summary>

**Correct Answer:** Workflow YAML file

**Explanation:** Workflow files in `.github/workflows/` specify `on:` triggers, `jobs:`, and `runs-on:` runner environments.
</details>

**Question 2:** In Azure Pipelines, what is the primary purpose of a Service Connection?
* [ ] To store source code
* [x] To authenticate securely to external services like Azure Cloud or Docker Registries
* [ ] To run unit tests
* [ ] To edit YAML syntax

<details>
<summary>🔑 Click to Reveal Answer & Explanation</summary>

**Correct Answer:** Option B

**Explanation:** Service Connections encapsulate service principals, OAuth tokens, or API keys required to connect Azure Pipelines to target deployment environments safely.
</details>
